Class LocalStream

本地媒体流

  • 由RTC视频房间对象ClientRoom创建；外部不能直接new构造
  • 可对本地媒体流进行初始化、发布、取消发布、暂停、恢复、替换媒体流（如切换摄像头）、关闭

Optional onWebRtcStateChanged

onWebRtcStateChanged: (newState: string) => void

Inherited from Stream.onWebRtcStateChanged

回调函数，媒体流传输状态发生变化

  • "new" ICE 代理正在搜集地址或者等待远程候选可用
  • "checking": ICE 代理已收到至少一个远程候选，并进行校验，无论此时是否有可用连接。同时可能在继续收集候选
  • "connected": ICE代理至少对每个候选发现了一个可用的连接，此时仍然会继续测试远程候选以便发现更优的连接。同时可能在继续收集候选
  • "completed": ICE代理已经发现了可用的连接，不再测试远程候选
  • "failed": ICE候选测试了所有远程候选没有发现匹配的候选。也可能有些候选中发现了一些可用连接
  • "disconnected": 测试不再活跃，这可能是一个暂时的状态，可以自我恢复
  • "closed": ICE代理关闭，不再应答任何请求

init

  • init(isWebcam: boolean, constraints: MediaStreamConstraints): Promise<void>

  • 初始化本地媒体流

    • 为了音频视频同步，一般同时初始化音频视频，而不是单独创建音频、视频的LocalStream对象

    Parameters

    • isWebcam: boolean

      视轨类型。true-表示摄像头；false-表示共享桌面

    • constraints: MediaStreamConstraints

      媒体约束。见WebRTC MediaStreamConstraints定义

    Returns Promise<void>

pause

  • pause(options?: { audio?: boolean; video?: boolean }): void

  • 暂停发布（静音）

    Parameters

    • Optional options: { audio?: boolean; video?: boolean }
      • options.audio==true，表示对音轨静音
      • options.video==true，表示对视轨静音
      • options==undefined或options=={audio:true,video:true}，表示同时对音轨、视轨静音
      • Optional audio?: boolean
      • Optional video?: boolean

    Returns void
